## Performance

**The headline wins this weekend are in this section.**

- **YOLO auto-accelerator** — CUDA → MPS → CPU picked in `load_model()`. Large CPU drop in multi-source demos; previously always CPU. — [6428bd6](../../)
- **Encode only when someone is watching** — `StreamSlot.has_viewers` counts active MJPEG subscribers and recent polls (`mark_polled`). Idle tiles stop burning JPEG encode cycles. — [79a50f3](../../), [da2e452](../../)
- **Auto-shedding policy** on the backend + **MJPEG decoder cap** on the frontend. Caps concurrent decoders so the browser doesn't melt at ≥6 tiles. — [f15b221](../../)
- **Transport split by protocol** — HTTPS tiles use MJPEG (one push-based connection per tile); HTTP tiles poll `/admin/frame/{id}` at ~400 ms. This works around the browser's 6-connections-per-host cap in local dev, where MJPEG+SSE would deadlock past ~4 tiles. — [7f938da](../../)
- **Detection toggle without stream interruption** — per-slot `detection_enabled` lets ops drop inference load on quiet tiles without restarting the stream. — [df478f2](../../)
- **Warm reload of `TARGET_FPS`** — tune frame rate live without bouncing the server. — [1d92da5](../../)
- **Optimistic UI + 8 s busy escape** — UI no longer blocks on server round-trips; stuck busy states auto-clear. — [286a492](../../)
- **ImpactCard polling 15 s → 5 s** — faster feedback on apply impact. — [e2832b5](../../)
